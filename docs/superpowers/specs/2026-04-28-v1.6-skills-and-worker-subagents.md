## 1. Goal

V1.5 shipped three live consulting types (`market_entry`, `pricing`, `profitability`) on a single shared 5-stage engine. Forensic analysis on the live database (614 runs, 798 evidence rows, 839 artifacts, 363 gates) surfaced **eleven** systemic findings that prevent V1.5 from delivering deep-research consulting at the quality bar advertised in the V1 plan.

**V1.6 closes those gaps via two interlocking changes:**

1. **Skills injection** — the 16 `backend/app/skills/*/SKILL.md` packs (488-line strategic-analysis, financial-modeling, etc.) are loaded into stage / synthesis / reviewer / audit prompts via a `SkillLoader`. Profiles declare which skills apply where.
2. **Worker subagents** — each stage fans out to 2–4 named worker subagents (e.g. `stage1_foundation.market_sizing`, `.customer`, `.regulatory`) executing in parallel, each producing its own artifacts and evidence. A merge step assembles a stage summary; the per-stage reviewer then evaluates the merged output.

The combination converts V1.5's "1 LLM call → 1–3 artifacts per stage" pattern into "1 LLM call per worker × N workers → 4–12 artifacts per stage with deeper, source-grounded reasoning."

## 3. Non-Goals

- No new consulting types beyond the existing 4 catalog rows (`ma`, `market_entry`, `pricing`, `profitability`).
- `ma` stays a stub (no real pipeline).
- No reviewer-overrides UI.
- No new web-search providers (existing tavily / exa / perplexity / duckduckgo registry).
- No frontend graph viz library swap (extend the custom SVG `AgentGraph`).
- No multi-user / role / billing changes — single-user local mode unchanged.
- No streaming token-by-token UI (still per-event SSE).
- No removing the synthesis self-heal — it stays as the last line of defence even after upstream evidence discipline tightens.

### 4.2 SkillLoader Contract

```python
# backend/app/agents/_engine/skills.py
from functools import lru_cache
from importlib import resources

_SKILL_PACKAGE = "app.skills"

@lru_cache(maxsize=64)
def load_skill(slug: str) -> str:
    """Read SKILL.md for `slug`, strip YAML frontmatter, return body.
    Raises FileNotFoundError if the slug doesn't exist."""

def render_skills_block(slugs: tuple[str, ...]) -> str:
    """Return '## Applicable Consulting Skills\n\n<concatenated bodies>' or '' for empty input."""

def inject_skills(system_prompt: str, slugs: tuple[str, ...]) -> str:
    """Prepend the skills block to system_prompt; passthrough if no slugs."""
```

- `lru_cache` is process-local; safe across coroutines (read-only).
- Frontmatter format: `---\n...\n---\n` — stripped via simple regex.
- Order is preserved as supplied.

All new fields default to empty → existing profiles compile unchanged. `validate()` is extended to read every referenced skill / per-stage reviewer prompt.

### 4.4 Worker Fanout

For a stage with `workers = (W1, W2, W3)`:

1. Build a per-worker user message containing:
   - the worker's system prompt (`profile.load_worker_prompt(stage_slug, worker_slug)`)
   - injected skills (`stage.required_skills + worker.required_skills`)
   - framing brief, scope, existing artifacts (same as stage today)
2. Bind `web_search` and `rag_search` tools to the worker model (ReAct loop, max 6 iterations, structured `StageOutput` finalization).
3. Run all workers in `asyncio.gather(...)` under `asyncio.Semaphore(4)`.
4. Each worker writes `<stage_slug>/<worker_slug>/<filename>` artifacts.
5. Merge step:
   - Union all worker `StageOutput.artifacts` and `evidence`.
   - Optionally write `<stage_slug>/_summary.md` with one-paragraph synopsis from each worker (no extra LLM call — assembled from `worker.summary`).
6. Reviewer evaluates the merged set against the per-stage prompt (with per-stage skills injected).

When `workers=()` (default), behaviour falls back to the V1.5 single-call path so unmodified profiles still work.

### 4.5 ReAct Stage Loop

Implemented inside `make_stage_node` (and reused by worker invocation):

- `model.bind_tools([web_search, rag_search])` — tools are run-scoped (built per-stage-invocation by a `tools_factory(run_id)` that V1.6 introduces).
- Loop:
  1. `model.ainvoke([System, Human])` → may emit `tool_calls`.
  2. For each tool call: invoke tool, append `ToolMessage`.
  3. Repeat until no `tool_calls` or 6 iterations reached.
  4. Final structured output call: `model.with_structured_output(StageOutput).ainvoke([all messages, "Now produce StageOutput"])`.
- Mandate: the stage records a counter; a worker that reaches the structured-output step having executed **zero** tool calls publishes an `agent_message` warning (`"<worker>: produced StageOutput without tool use"`) — recorded as soft signal, not a failure (so scripted-test models still work).

### 4.6 Per-Stage Reviewer

Prompt resolution order:
1. `profile.reviewer_prompt_for_stage[stage_slug]` if set
2. fallback to `profile.reviewer_prompt`

Skill injection: `profile.reviewer_skills_per_stage.get(stage_slug, ())`.

### 4.7 Run Lifecycle

`Run` gains 3 nullable columns:
- `started_at` — set when worker begins `continue_after_framing`
- `completed_at` — set when status transitions to `completed`/`failed`/`cancelled`
- `heartbeat_at` — bumped every 30s by a background heartbeat task

Recovery (called on app startup + before each new spawn):
- Find rows where `status='running' AND heartbeat_at < now() - 5 minutes`.
- Mark as `failed`, set `completed_at=now()`, publish `system.run_failed` with `reason="staleness: no heartbeat for >5 minutes"`.

Per-run timeout:
- `RUN_TIMEOUT_SECONDS=7200` (120 min).
- Heartbeat task also enforces: if `now() - started_at > timeout`, cancel the run task and mark failed.

### 4.8 Scripted-Model Production Guard

`get_run_model_factory_builder` in `app/api/runs.py` returns the production builder; tests override via `app.dependency_overrides`. The API never accepts `model_factory` from the request body (already true), and `default_model_factory` raises if it ever encounters a sentinel marker. We document the test fixture pattern in `backend/tests/conftest.py`.

### 4.9 Worker SSE Events

- Worker `agent` field: `"<stage_slug>.<worker_slug>"` (e.g. `"stage1_foundation.market_sizing"`).
- Stage merge step still emits `agent="<stage_slug>"` for the stage-level summary.
- Existing event types unchanged; only `agent` field gains the dotted child id.
- Frontend `useAgentStates` parses `parent.child`, attaches the child to the parent's `children: AgentNodeData[]` list, and `AgentGraph` renders them as collapsible mini-nodes below each stage.

### 4.10 Path Normalization

Synthesis + audit use a path normalizer that accepts known synonyms (`stage_3_risk` ↔ `stage3_risk`, `risk.md` ↔ `findings.md`) and surfaces all artifacts into the synthesis context. The artifact table keeps original paths; this is a read-side reconciliation only.

### 4.11 Backfill

`scripts/backfill_stale_runs.py` (one-off):
- For all rows where `status='running'` and `created_at < now() - 24h`:
  - Set `status='failed'`, `completed_at=now()`.
  - Insert `events` row `type='system.run_failed'` payload `{"rationale": "backfill: orphaned by V1.5 worker"}`.
- Idempotent (skips already-failed).

## 10. Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Worker fanout × 3 increases per-run cost ~3× | `worker_concurrency=4` semaphore caps parallel calls; budget tracker still applies; optional disable via `workers=()`. |
| Long context from injected skills bloats prompts | `lru_cache` keeps file reads cheap; per-call token cost capped by skill pack sizes (largest = strategic-analysis ≈ 4 KB); no profile assigns more than 2 skills per stage initially. |
| ReAct loop hangs on misbehaving model | `react_max_iterations=6` hard cap; per-run timeout safety net. |
| Scripted-model tests break on tool-bind | `FakeChatModel` extended with `tool_calls` script; existing tests pass tools=() and continue to work. |
| Heartbeat task races with cooperative cancel | Heartbeat is a `asyncio.create_task` cancelled in `finally:` of `continue_after_framing`. |
| Per-stage reviewer prompts diverge from existing rubric | Each new prompt explicitly extends the shared rubric ("Apply the standard rubric, plus the following stage-specific checks: …"). |

## 11. Out of scope for V1.6 (deferred to V1.7+)

- Real `ma` pipeline.
- Reviewer overrides UI / human-in-the-loop gate intervention.
- Multi-turn worker collaboration (workers reading each other's drafts inside a stage).
- Persistent skill cache invalidation (skills change → restart suffices for now).
- Cross-run learning from rejected drafts.
