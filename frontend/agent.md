# Frontend — agent.md

Next.js 16 (App Router) UI for the Consulting Research Agent. Talks to the FastAPI backend in `backend/` over REST + SSE.

## How to use this system

```bash
cd frontend
pnpm install
pnpm dev          # local dev server on :3000 (Turbopack)
pnpm build        # production build
pnpm start        # run production build
pnpm lint         # eslint
pnpm typecheck    # tsc --noEmit
```

If port 3000 is taken: `pnpm exec next start -p 3100` (or `pnpm dev -p 3100`).

> **Heads-up:** This project uses Next.js 16 (Turbopack default, React 19). APIs and conventions differ from older Next.js versions. The auto-generated `frontend/AGENTS.md` and `frontend/CLAUDE.md` instruct agents to consult `node_modules/next/dist/docs/` before writing Next-specific code. Follow that.

## Project overview

Single-page-ish web app where a consultant picks a task, fills a brief / answers a clarifying questionnaire, kicks off an agent run, and watches a live trace + final report stream in. Backend (FastAPI + LangGraph + DeepAgents) does all the heavy lifting; frontend is presentation, control, and SSE consumer.

## Conventions

- TypeScript strict (per `tsconfig.json`)
- TailwindCSS v4 (PostCSS-based, no `tailwind.config.*`); theme tokens live in `app/globals.css`
- shadcn/ui for UI primitives (style `base-nova`, neutral base, CSS variables, base-ui under the hood)
- pnpm for package management
- Server components by default; add `"use client"` only when needed (state, effects, browser APIs)
- API access goes through `lib/api.ts`, never inline `fetch` in components
- SSE consumption goes through `lib/sse.ts` (planned M5.7)
- Path alias: `@/*` → repo `frontend/` root
- No per-component `agent.md` files — this single file is the registry
